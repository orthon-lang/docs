# Execution Image

> **SUPERSEDED — This concept has been superseded by the Execution Program model.**
>
> See [`EXECUTION_PROGRAM.md`](./EXECUTION_PROGRAM.md) for the current version.
>
> The Execution Image is now understood as a *materialisation format* —
> one of several possible outputs produced by an Execution Engine
> (specifically, the OCI Builder, MicroVM Builder, or AOT Compiler).
> It is no longer the central abstraction.
>
> This file is retained for reference and will be removed after
> the Concept Design Review (Milestone 2).
